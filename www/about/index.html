Welcome to Webserver 42

Introduction

Webserver 42 is a project developed at the 42 school, designed to teach students the core concepts of web servers. The goal is to create a lightweight and functional web server capable of handling basic HTTP requests and responses.

Features

  • Handles HTTP GET, POST, and DELETE requests.
  • Supports serving static files like HTML, CSS, and JavaScript.
  • Custom error pages for various HTTP status codes.
  • Ability to configure ports and server root directory.

Why Webserver 42?

The Webserver 42 project pushes students to understand how web servers function behind the scenes. By building the server from scratch, students gain insight into:

  • Socket programming and network communication.
  • HTTP protocol structure and functionality.
  • Handling concurrent client connections.
  • Efficient resource management and error handling.

Steps to Build Your Webserver

  1. Set up your environment and initialize your project.
  2. Learn the basics of sockets and how to accept connections.
  3. Implement parsing and handling HTTP requests.
  4. Add support for serving static files and dynamic content.
  5. Test your server with tools like cURL or Postman.

Conclusion

Webserver 42 is not just a project; it's a journey into the backbone of the internet. Completing it will leave you with a solid understanding of how servers work and prepare you for more advanced topics in network programming and web development.
